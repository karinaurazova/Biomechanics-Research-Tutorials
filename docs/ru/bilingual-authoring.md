# Подготовка билингвальных материалов

Каждый новый tutorial представляет собой один исполняемый проект с двумя языковыми слоями учебных материалов.

## Рекомендуемая последовательность

1. Зафиксировать исследовательский вопрос, уравнения, параметры и границу доказательности.
2. Реализовать и верифицировать общий код Python.
3. Получить один набор численных результатов и сгенерировать из тех же скриптов локализованные английские и русские рисунки.
4. Подготовить английскую учебную версию.
5. Создать русскую версию с эквивалентным научным смыслом.
6. Перевести Markdown и локализованные сообщения notebook, сохранив вычислительно эквивалентные ячейки кода.
7. Подготовить парные материалы мастер-класса и оценивания.
8. Выполнить `pytest -q` для проверки языковой полноты и численного поведения.

## Контрольный список релиза

- [ ] Существуют `README.md` и `README.ru.md`.
- [ ] Каждая глава представлена на двух языках.
- [ ] Каждое задание представлено на двух языках.
- [ ] Английский и русский notebooks используют один численный алгоритм.
- [ ] Методические заметки и материалы мастер-классов имеют пары.
- [ ] Каждый включённый в репозиторий учебный рисунок и GIF имеет локализованную пару.
- [ ] Парные визуализации построены по одинаковым параметрам и данным.
- [ ] Уравнения, параметры и границы научных утверждений совпадают.
- [ ] Все локальные ссылки разрешаются.
- [ ] Воспроизводимость обеспечивается одной общей кодовой базой.

## Принцип перевода

Переводится смысл, а не порядок слов. Следует использовать устоявшуюся русскую терминологию, сохранять стандартные идентификаторы кода и не усиливать утверждения в одной из языковых версий.
