# Как устроен репозиторий

[English](https://github.com/karinaurazova/Biomechanics-Research-Tutorials/blob/main/docs/how-the-repository-works.md)

## Вычислительная цепочка

```text
параметры и допущения
        ↓
общая модель в src/biomechanics_tutorials
        ↓
массивы NumPy с синтетическими численными результатами
        ↓
notebook: интерактивные графики внутри документа
или
скрипт эксперимента: сохранённые PNG/GIF-файлы
```

## Для чего нужен notebook

Notebook — это **интерактивный учебный маршрут** по tutorial. Его ячейки:

- вводят исследовательский вопрос и допущения;
- импортируют общую локальную модель;
- задают параметры;
- запускают численные вычисления;
- показывают промежуточные значения и графики;
- позволяют изменять параметры и интерпретировать результат.

Графики в notebook вычисляются при выполнении ячеек. Notebook не является простым просмотрщиком изображений из папки `figures/`.

## Откуда появляются сохранённые рисунки

Файлы в `figures/` и `animations/` — результаты воспроизводимости, создаваемые скриптами из папки `experiments/`. Они хранятся в репозитории, чтобы пользователь GitHub мог увидеть результаты без запуска Python. Любой файл можно пересоздать.

Например:

```text
tutorials/01-fiber-reorientation/experiments/baseline.py
        ↓
tutorials/01-fiber-reorientation/figures/baseline.png
tutorials/01-fiber-reorientation/figures/baseline_ru.png
```

Notebook и скрипты экспериментов используют одни и те же общие математические функции. Поэтому интерактивный разбор и сохранённые результаты относятся к одной модели, хотя компоновка графиков может различаться.

## Откуда берутся численные значения

В текущих tutorials используются явно заданные **синтетические параметры**. Скрытые датасеты не загружаются. Массивы времени, ориентации, растяжения, плотности вероятности и напряжения вычисляются в памяти по уравнениям, описанным в учебных главах.

## Три способа работы

1. **Только чтение:** открыть README и готовые рисунки на GitHub.
2. **Интерактивное обучение:** выполнить notebook и изменить параметры.
3. **Полная воспроизводимость:** запустить `reproduce.py` или отдельные скрипты и заново создать все рисунки и анимации.
