# How the repository works

[Русский](ru/how-the-repository-works.md)

## The computational flow

```text
parameters and assumptions
        ↓
shared model in src/biomechanics_tutorials
        ↓
NumPy arrays with synthetic numerical results
        ↓
notebook: interactive inline plots
or
experiment script: saved PNG/GIF outputs
```

## What the notebook is for

A notebook is the **interactive educational route** through a tutorial. Its cells:

- introduce the question and assumptions;
- import the shared local model;
- define parameters;
- run numerical calculations;
- display intermediate values and plots;
- invite the learner to change parameters and interpret results.

The notebook computes its plots when cells are executed. It is not merely a viewer for images stored in `figures/`.

## Where the committed figures come from

The files in `figures/` and `animations/` are reproducibility outputs generated by scripts in `experiments/`. They are committed so that GitHub readers can see results without executing Python. They can always be regenerated.

For example:

```text
tutorials/01-fiber-reorientation/experiments/baseline.py
        ↓
tutorials/01-fiber-reorientation/figures/baseline.png
tutorials/01-fiber-reorientation/figures/baseline_ru.png
```

The notebook and experiment scripts call the same shared mathematical functions. Therefore, the educational walkthrough and saved outputs represent the same model, even though their layouts can differ.

## Source of the numbers

Current tutorials use explicitly defined **synthetic parameters**. No hidden dataset is loaded. Arrays such as time, orientation, stretch, probability density, and stress are calculated in memory from equations documented in the tutorial chapters.

## Three ways to use the repository

1. **Read only:** view README files and committed figures on GitHub.
2. **Interactive learning:** run a notebook and change parameters.
3. **Full reproduction:** run `reproduce.py` or the experiment scripts to rebuild all figures and animations.
