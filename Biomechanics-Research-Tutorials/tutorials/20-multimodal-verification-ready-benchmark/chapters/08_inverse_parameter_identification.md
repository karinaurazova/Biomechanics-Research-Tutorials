# Inverse parameter identification

The inverse problem is written as A p approximately equals y. The matrix A is assembled from strain fields and recovered structural descriptors; y contains force observations; p contains material parameters.

Four scenarios are compared: oracle ground truth, SEM-only, polarization-only and multimodal fusion. The oracle case checks the mechanical inverse problem itself, while modality-based cases test image-to-mechanics error propagation.

A low residual can still hide biased parameters because structural errors can be absorbed by stiffness-like coefficients. The benchmark therefore reports both force residual and parameter error.

The chapter should be read together with the code. Every quantity mentioned here is created explicitly in `multimodal_benchmark.py` and regenerated by `reproduce.py`. The purpose is not to hide complexity behind a black box, but to show which assumptions enter the pipeline.

For interpretation, always ask four questions: what is the hidden truth, what is observed, what is reconstructed, and which metric compares them? This four-part logic prevents a visually appealing result from being mistaken for a verified mechanical conclusion.

## Implementation details

The inverse problem assembles one linear system per structural scenario. The oracle scenario uses exact orientation, density and connectivity. SEM-only and polarization-only scenarios use recovered fields. The fusion scenario uses a combined structural estimate. All scenarios see the same load cases and force observations, so differences in parameters can be interpreted as consequences of structural recovery.

## What to check while reading

Open `parameter_recovery.csv` and compare relative force residual with relative parameter error. A small residual with a large parameter error indicates compensation. The solver found parameters that reproduce the observed forces, but those parameters are not the true ones. That distinction is one of the main lessons of the module.

## Typical misunderstanding

Fitting the measured force is not the same as identifying the material. In inverse mechanics, errors in geometry, orientation, density or boundary conditions can be absorbed into material coefficients. Therefore parameter identification must always be interpreted together with structural accuracy and identifiability diagnostics.
