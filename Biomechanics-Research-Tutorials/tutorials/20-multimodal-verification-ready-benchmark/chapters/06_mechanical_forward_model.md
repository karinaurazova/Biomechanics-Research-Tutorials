# Mechanical forward model

The forward model uses small-strain plane-stress kinematics and a linear-in-parameters anisotropic stress law. It includes matrix stiffness, density gain, fiber scale and connectivity gain.

The structure fields define the design matrix, while the unknown parameters scale the stress. This separation makes the inverse problem transparent and exactly checkable.

Multiple load cases are used because one loading mode rarely identifies all parameters. Shear and transverse loading improve separation between matrix and fiber contributions.

The chapter should be read together with the code. Every quantity mentioned here is created explicitly in `multimodal_benchmark.py` and regenerated by `reproduce.py`. The purpose is not to hide complexity behind a black box, but to show which assumptions enter the pipeline.

For interpretation, always ask four questions: what is the hidden truth, what is observed, what is reconstructed, and which metric compares them? This four-part logic prevents a visually appealing result from being mistaken for a verified mechanical conclusion.

## Implementation details

The model is linear in four parameters but nonlinear in the sense that its basis depends on spatial structure. The isotropic part responds to `exx`, `eyy` and `gxy` through a plane-stress matrix-like term. The fiber part projects strain along the local fiber axis and returns a stress contribution aligned with that axis. Density and connectivity scale these basis terms before the unknown parameters are estimated.

## What to check while reading

Follow the dimensions of the design basis. At every pixel, the code builds a small matrix mapping four parameters to three stress components. Averaging those stress components over the specimen gives global force-like equations. This is a simplified analogue of assembling a forward mechanical model and then using it inside an inverse problem.

## Typical misunderstanding

Because the inverse equation is linear in parameters, it may look too simple. The purpose is not to replace nonlinear soft-tissue mechanics. The purpose is to make the image-to-parameter logic auditable. Once the logic is clear, the same benchmark structure can be extended to nonlinear finite-element or RVE-based models.
