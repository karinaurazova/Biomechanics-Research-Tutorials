# How to extend the benchmark

Every stage can be replaced while the rest of the pipeline stays fixed. This makes the module useful for student projects and method comparison.

Possible extensions include stronger microscopy artifacts, 3D stacks, learned segmentation, SAM/μSAM prompts, ODF reconstruction, pore-size metrics, nonlinear mechanics, inverse finite elements and Bayesian calibration.

A good extension must define four things: ground truth, observation, reconstruction and metric. Without these, the pipeline becomes difficult to interpret.

The chapter should be read together with the code. Every quantity mentioned here is created explicitly in `multimodal_benchmark.py` and regenerated by `reproduce.py`. The purpose is not to hide complexity behind a black box, but to show which assumptions enter the pipeline.

For interpretation, always ask four questions: what is the hidden truth, what is observed, what is reconstructed, and which metric compares them? This four-part logic prevents a visually appealing result from being mistaken for a verified mechanical conclusion.

## Implementation details

The functions are intentionally replaceable. A new segmentation method only needs to output a mask with the same shape. A new orientation method only needs to output an axial angle field. A new mechanical law needs to provide a forward response and an inverse strategy. This makes the tutorial a scaffold for larger projects.

## What to check while reading

Before adding a sophisticated method, define the baseline it must beat and the metric it should improve. For example, a SAM/μSAM workflow should not only improve mask appearance; it should improve continuity, orientation support or parameter recovery. A Bayesian inverse method should not only return intervals; those intervals should react sensibly to noise and load-case changes.

## Typical misunderstanding

Adding complexity does not automatically improve science. A more advanced model is useful only if it answers a clearer question, reduces bias, improves identifiability or quantifies uncertainty better than the transparent baseline.
